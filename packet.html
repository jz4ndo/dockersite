Docker Compose

Un tool che permette la definizione e l'esecuzione di applicazioni multi-container, può essere definito come un orchestratore che però al contrario di Kubernetes e Swarm non può gestire host multipli.

Docker Compose viene utilizzato sempre più spesso dalle aziende sopratutto per l'esecuzione di app che necessitano di un back-end e di un front-end, per la realizzazione verranno quindi usati vari servizi, per esempio: Angular, Flask, MySQL.

Per ognuno di questi servizi va compilato un Dockerfile con le caratteristiche che definiscono il servizio stesso. Tutti i vari Dockerfile, con l'avvio di docker compose, verranno trasformati in immagini, le quali verranno gestite in ambienti isolati.

Per eseguire un'app multi-container occorre la stesura di un file di configurazione chiamato: docker-compose.yaml, questo file si riferisce ai vari Dockerfile presenti nel progetto.

File Compose YAML

version: "3.5"
services:
backend:
build:
context: ./backend
dockerfile: Dockerfile
ports:
- "3000:3000"
links:
- mongo
mongo:
image: mongo
volumes:
- ./db_vol/:/data/db
ports:
- "27018:27017"

Compose file

Tramite Compose si possono creare tutti i servizi di una certa configurazione tramite un solo comando. Il file Compose si riferisce ai Dockerfile che sono presenti nel progetto, in seguito poi la tecnologia di Compose realizza per noi (in qualche secondo) tutte le immagini necessarie.

Come si vede dall'esempio, nel file yaml vanno specificati vari campi come:

Version, Context, Dockerfile, Ports, Links, Volumes

Dopo aver specificato questi campi si può procedere con l'avvio di Docker Compose, tramite il comando:

docker-compose up

E per fermare Compose si usa il comando:

docker-compose down
