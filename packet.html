Docker in azienda

In azienda viene usato sempre più spesso Docker compose; è uno strumento per l'esecuzione di app multi-container. Grazie a un file Compose si creano e si avviano tutti i servizi della propria configurazione.

Per esempio se si vuole creare una applicazione web si dovranno utilizzare vari servizi, come: Angular (Frontend), Flask (Backend), MySQL (Database).

Ogni servizio ha bisogno di un Dockerfile, che viene buildato per creare l'immagine, per far si che avvenga tutto questo va creato un file compose che desciva le nostre esigenze.

File Compose YAML

version: 3
services:
angular:
build:
context: angular
ports: - 5000:5000
flask:
build:
context: flask
volumes: ./flask/data:/data
mysql:
image: mysql:5.7
restart: always
ports:
- 27017:27017
environment:
MYSQL_ROOT_USERNAME:root
MYSQL_ROOT_PASSWORD:example
volumes:
mysql-data-volume:/dt/db

Compose file

Esso builda i Dockerfile e le corrispettive immagini, in questo file vanno elencati i servizi che vogliamo utilizzare nell'applicazione.
