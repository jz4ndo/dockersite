Cos'è Docker?

Docker è una piattaforma software che ci permette di creare, sviluppare e distribuire applicazioni molto rapidamente.

Docker utilizza degli ambienti isolati di sviluppo chiamati Container. Questi ambienti leggeri e veloci permettono a sviluppatori e amministratori di creare, spedire e eseguire app a basso costo e su qualsiasi scala.

Architettura Docker

Docker è basato su una architettura Client - Server.

Il Client prende il nome di Client Docker e il Server prende il nome di Daemon, essi comunicano tra loro tramite API REST.

Il Daemon ascolta le richieste e gestisce immagini, container, reti e volumi. Il Client invece è colui che esegui i comandi e ordina al Daemon di compiere certe azioni necessarie (build, run, stop, remove).

Oggetti Docker

In questo universo virtuale esistono vari "oggetti" che rendono possibile la magia di Docker.

Le immagini sono modelli di sola lettura per la creazione dei containers.

I container sono istanze eseguibili di una certa immagine, si possono creare, avviare, arrestare, spostare ed eliminare.

I registry sono luoghi comuni a tutti gli utenti in cui vengono memorizzate immagini già pronte per essere scaricate e usate (Docker Hub).

Dockerfile

FROM node:12-alpine
RUN apk add --no-cache python g++ make
WORKDIR /app
COPY . .
RUN yarn install --production
CMD ["node", "src/index.js"]

Dockerfile

Il dockerfile è un file di testo che contiene varie info fondamentali per la creazione dell'immagine.

Nel dockerfile vanno specificate vari dati, come il linguaggio di programmazione che vogliamo usare per l'immagine e qual è la directory del nostro progetto.

Questo file verrà tradotto da Docker Engine e verrà trasformato nell'immagine che necessitiamo.
