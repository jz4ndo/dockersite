Iniziare con Docker

Docker è un Container Software Platform Suite , cioè un prodotto creato per semplificare l'esecuzione e lo sviluppo di applicazioni.

La tecnologia di Docker permette ai developer di creare, testare e distribuire applicazioni con un'alta rapidità, abbattendo notevolmente costi e complicazioni.

L’idea della containerizzazione è proprio quella di permettere a una data applicazione di essere eseguita su qualsiasi tipo di sistema, dato che tutte le sue dipendenze sono già incluse nel container stesso.

Docker vs Virtual Machine

Quando Docker fu rilasciato nel 2013 venne paragonato alle classiche macchine virtuali, in realtà Docker ha un approccio totalmente diverso.

Le macchine virtuali emulano l'hardware, e il numero di macchine virtuali è sicuramente limitato mentre si potrebbero avere centinaia di container sulla stessa macchina.

Docker infatti viene eseguito direttamente sul sistema operativo, tutti i container condividono lo stesso hardware e kernel della macchina ospitante. Queste caratteristiche rendono i container veloci e leggeri.

Architettura Docker

Docker lavora secondo un'architettura Client - Server, i componenti prendono i nomi di:

Docker-Client: componente client che invia comandi e richieste al server

Docker-Daemon: componente server che gestisce i container e esegue le richieste del client.

Solitamente client e server si trovano sulla stessa macchina, il server si occupa dei container locali e l'utente interagisce attraverso il client. Il collegamento tra client e server sulla stessa macchina avviene tramite un socket Unix: /var/run/docker .sock

Tooling

Docker è stato ideato per Linux, ma può essere utilizzato anche su altri sistemi operativi come Windows, i componenti principali da tenere in considerazione sono:

Dockerfile: è la ricetta dell'immagine, un file di testo che ci permette di esprimere le personalizzazioni che vogliamo apportare all'immagine di partenza.
Image: è il risultato della compilazione del Dockerfile.
Container: è un'istanza di realizzazione di un'immagine.

Installazione

Docker può essere utilizzato a riga di comando oppure tramite l'apposito Docker Desktop, il software che ci permette di visuallizare le immagini e i container in esecuzione.

Per utilizzare Docker durante lo sviluppo delle applicazioni sul proprio IDE si consiglia di installare l'apposita estensione.

Dockerfile

FROM nginx:latest
WORKDIR /usr/share/nginx/html
COPY . .
EXPOSE 80

Utilizzo di Docker

Dopo aver completato l'installazione si procede con la creazione del progetto in questione tramite il proprio IDE. Con il completamento dell'applicazione si può procedere con la containerizzazione, questa fase è composta da vari passaggi.

Si continua con la creazione del Dockerfile nella directory del progetto, questo file specifica le caratteristiche dell' immagine che andremo a creare.

Il Dockerfile verrà tradotto e trasformato nell'immagine, l'immagine in seguito può essere visualizzata sulla dashboard di docker desktop.

Creare un'immagine

Dopo aver compilato il Dockerfile, per creare l' immagine basta eseguire un semplice comando nel terminale del nostro IDE.

$ docker build -t elaborato .

Il comando riportato sopra dice a Docker di cercare ed eseguire il Dockerfile che si trova nella directory corrente (.), il quale contiene tutte le specifiche dell'immagine.

Una volta eseguito verrà buildata un'immagine chiamata "elaborato", è come un vero e proprio sistema operativo che contiene il nostro progetto.

Avviare un Container

Dopo la creazione dell'immagine, per avviare il container basta eseguire un banale comando simile a quello che è stato usato prima.

$ docker run -d --name container1 -p 80:80 elaborato

Questo comando crea un container contenente la nostra immagine, grazie al mapping 80:80 il progetto sarà visibile sul browser tramite localhost:80

Condividere l'immagine

$ docker build -t elaborato .
